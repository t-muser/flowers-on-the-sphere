# Cahn-Hilliard Phase Separation on the Sphere

**One-line description.** A parametric ensemble of Cahn-Hilliard
phase-separation trajectories on the 2-sphere, started from seeded
small-amplitude Gaussian noise around a prescribed mean composition and
evolved through spinodal decomposition into the curvature-driven coarsening
regime.

**Extended description.** Each trajectory solves the mobility-form
Cahn-Hilliard equation on a thin radially-extruded shell mesh of a sphere
of radius `R = 5`. The free energy is the standard symmetric double-well
`F(φ) = φ²(1 − φ)²/4` with order parameter `φ ∈ [0, 1]` and an interface
penalty `(ε²/2)|∇φ|²`. The initial condition is i.i.d. Gaussian noise of
mean `mean_init` and variance `0.01` per cell (seeded from a `numpy`
`Generator`), so each trajectory is a deterministic function of the triple
`(ε, mean_init, seed)`. From this seed the dynamics produce three
recognisable regimes: linear growth of the dominant unstable wavelength,
sharp interface formation between the `φ ≈ 0` and `φ ≈ 1` phases, and
coarsening driven by interface curvature on the sphere. Unlike Galewsky
(deterministic rollup from a balanced jet) or Mickelin (forced active
turbulence), the Cahn-Hilliard system is dissipative and approaches a
geodesic-curvature-minimising domain pattern as `t → ∞`.

## Mathematical framework

The governing equation is the mobility-form Cahn-Hilliard equation on the
sphere `S²`,

$$
\partial_t \varphi \;=\; \nabla\!\cdot\!\bigl(M(\varphi)\,\nabla\mu\bigr),
\qquad
\mu \;=\; \frac{dF}{d\varphi} \;-\; \varepsilon^2 \nabla^2\varphi,
$$

with double-well bulk free energy density

$$
F(\varphi) \;=\; \tfrac{1}{4}\,a^2\,\varphi^2(1-\varphi)^2
\quad\Longrightarrow\quad
\frac{dF}{d\varphi} \;=\; a^2\,\varphi(1-\varphi)(1-2\varphi),
$$

and concentration-dependent mobility

$$
M(\varphi) \;=\; D\,a^2\,\bigl(1 - 6\varphi(1-\varphi)\bigr)
\;-\; D\,\varepsilon^2 \nabla^2\!\!.
$$

In FiPy's two-`DiffusionTerm` form this is implemented as

$$
\partial_t \varphi
\;=\; \nabla\!\cdot\!\bigl[D\,a^2(1 - 6\varphi(1-\varphi))\,\nabla\varphi\bigr]
\;-\; \nabla\!\cdot\!\bigl[D\,\varepsilon^2\,\nabla(\nabla^2\varphi)\bigr],
$$

i.e. a degenerate-mobility transport term plus a fourth-order interface
penalty. The PDE conserves the spatially-averaged composition `⟨φ⟩` and
monotonically decreases the total free energy `∫(F + (ε²/2)|∇φ|²)` to
within discretisation error.

### Linear instability and interface width

Linearising about the spatially-uniform state `φ = m`, modes of wavenumber
`k` grow at rate

$$
\sigma(k) \;=\; -D\,k^2\,\bigl[a^2(1 - 6m(1-m)) + \varepsilon^2 k^2\bigr],
$$

so the homogeneous state is **spinodally unstable** when
`m(1 − m) > 1/6`, i.e. for `0.211 < m < 0.789`. All three values of
`mean_init` in this dataset (`0.35`, `0.50`, `0.65`) sit inside that
window; the symmetric value `0.50` gives the strongest instability and the
highest equilibrium interface coverage. The fastest-growing wavelength
scales as `λ_max ∼ ε/√(|a²(1 − 6m(1 − m))|)`, so smaller `ε` yields
finer initial spinodal patterns.

### Fixed non-dimensional scales

| Symbol | Value | Meaning |
| --- | --- | --- |
| `R`     | `5`     | Sphere radius (non-dimensional) |
| `D`     | `1`     | Mobility scale |
| `a`     | `1`     | Bulk-energy scale (sets the `[0, 1]` minima of `F`) |
| `var₀`  | `0.01`  | IC noise variance |

Time is reported in solver units; with `D = a = 1` the natural time
scale is `ε⁻¹` for the slowest unstable mode and `ε²/D = ε²` for
interface relaxation, so coarsening dynamics for different `ε` values
proceed at different effective speeds across the ensemble.

## Data specifications

### Grid

- **Discretization.** Regular `(lat, lon)` grid with `N_lat = 256`,
  `N_lon = 512`, equispaced and pole-excluding. Latitude centres
  `lat_k = -π/2 + (k + 0.5)·π/N_lat` in radians, stored as
  `degrees_north`. Longitude centres `lon_k = k·2π/N_lon` in `[0, 2π)`,
  stored as `degrees_east`.
- **Native solver mesh.** Unstructured triangulated spherical surface
  generated by [Gmsh](https://gmsh.info/) as a stitched 6-patch
  parameterisation, then radially extruded to a one-cell-thick shell
  (`r → 1.1·r` per `Mesh.extrude`) so face normals are unambiguous.
  At the production target cell size `0.3` this yields ≈ 9 000 cell
  centres in `ℝ³`. Cell centres are stored once per run as
  `/scales/cell_xyz`.
- **Resampling.** Cell centres are projected back to the unit sphere,
  a single `scipy.spatial.cKDTree` is built, and the four nearest cell
  centres of every regular `(lat, lon)` grid point are queried once.
  Each snapshot is then formed by inverse-distance-weighted averaging of
  those four cell values onto the regular grid (chord-distance handles
  the longitude wrap-around natively, and pulling from multiple cells
  averages out the inner/outer duplicate sampling that comes from the
  radial extrusion).

### Temporal layout

- **Snapshot cadence.** Every `10` solver-time units.
- **Simulation length.** `1 000` solver-time units.
- **Output window.** Full window kept; no spinup is discarded (the
  spinodal decomposition itself is part of the dataset). The first
  snapshot at `t = 0` is the seeded Gaussian noise IC.
- **Trajectory shape.** 101 snapshots per trajectory, `10` solver units
  apart.
- **Time coordinate.** Stored as solver units (`float64`, see
  the `time_units` Zarr attribute).
- **Time-step schedule.** Internally, an exponentially growing time step
  `dt = min(max_dt, exp(dexp))` with `dexp += 0.01` per accepted step
  (the canonical FiPy CH schedule), starting from `dexp = -5` and
  capped at `max_dt = 100`. Snapshot times are recorded after each
  step and may overshoot the requested cadence by up to one `dt`,
  hence small irregularities in the on-disk `time` coordinate.

### Available fields

| Field name | Units            | Description |
| ---        | ---              | --- |
| `phi`      | dimensionless    | Order parameter (composition / phase indicator), `≈ 0` and `≈ 1` are the two coexisting phases |

Stored as `float32`. Small transient overshoots outside `[0, 1]` are
expected during interface formation (typically `−0.02 ≲ φ ≲ 1.02`); they
relax back as interfaces sharpen.

### Dataset size

- **Number of trajectories.** 48.
- **Shape per trajectory.** `(time=101, field=1, lat=256, lon=512)`.
- **Per-trajectory size.** ≈ 53 MB (float32, uncompressed).
- **Total ensemble size.** ≈ 2.5 GB.
- **Consolidated shape.** `(run=48, time=101, field=1, lat=256, lon=512)`.

### Storage format

Per-run Zarr v3 stores at
`…/cahn-hilliard-sphere/processed/run_XXXX.zarr`, chunked as
`(time=1, field=1, lat=256, lon=512)` for fast per-snapshot random
access. Raw FiPy HDF5 files are written single-file per run
(`<out_dir>/cahn_s1.h5`, with `/scales/cell_xyz`, `/scales/sim_time`,
and `/tasks/phi`) and discarded after resampling. A consolidated
`dataset.zarr` stacks all runs along a leading `run` dimension with
parameter arrays carried as `param_*` coords. The `manifest.json` at
the dataset root carries the full list of `(run_id, param_hash, params)`
tuples.

## Initial conditions

Each run is seeded with a per-cell i.i.d. Gaussian noise field

$$
\varphi_0(\mathbf{x}_c) \;=\; m \;+\; \xi_c,
\qquad \xi_c \sim \mathcal{N}(0, \mathrm{var}_0)\ \text{i.i.d.},
$$

over the cell centres of the unstructured mesh, where `m = mean_init`
and `var₀ = 0.01`. The noise is drawn from
`numpy.random.default_rng(seed).normal(...)` rather than FiPy's
`GaussianNoiseVariable` (which has no seed parameter), so each trajectory
is bit-reproducible from `(seed, mean_init, mesh)`. Conservation of the
spatial mean by the CH evolution then implies `⟨φ⟩(t) = m` for all `t`
up to discretisation error.

This keeps each trajectory a deterministic function of the triple
`(ε, mean_init, seed)`.

## Physical setup

- **Domain.** Full sphere `S²`, radius `R = 5`, no boundary. Implemented
  as a thin radial shell so face normals are unambiguous; the inner and
  outer faces are treated identically by the resampler via the
  unit-sphere projection of their cell centres.
- **Boundary conditions.** None — the closed surface has no boundary.
- **Mass conservation.** The mobility form conserves `∫_S² φ dA`
  exactly in the continuous problem; the FiPy discretisation conserves
  it to within solver tolerance.
- **Timestepper.** First-order implicit Euler with FiPy's default linear
  solver, single iteration per step (no inner sweep loop).
- **Time-step ramp.** `dt_n = min(max_dt, exp(dexp_n))` with
  `dexp_0 = -5`, `dexp_{n+1} = dexp_n + 0.01`, `max_dt = 100`. The
  ramp is shared across all trajectories so the on-disk time
  coordinates align across runs to within rounding.
- **Integration window.** `0 ≤ t ≤ 1 000`. All 101 snapshots are kept;
  the IC at `t = 0` is included.

## Parameter space

Runs are laid out on an explicit tensor grid over three axes
(`4 · 3 · 4 = 48` runs). Runs are indexed `run_0000 … run_0047` in
row-major order over the tuple `(epsilon, mean_init, seed)`.

| Parameter            | Symbol      | Values                         | Count |
| ---                  | ---         | ---                            | ---   |
| Interface width      | `epsilon`   | 0.5, 1.0, 1.5, 2.0             | 4     |
| Initial composition  | `mean_init` | 0.35, 0.50, 0.65               | 3     |
| IC seed              | `seed`      | 0, 1, 2, 3                     | 4     |

Held fixed across the ensemble: `D = 1`, `a = 1`, `variance = 0.01`,
`radius = 5`, mesh `cell_size = 0.3`, snapshot cadence `10`, simulation
length `1 000`, time-step schedule.

The `seed` axis is the IC randomisation factor (the noise field is i.i.d.
across cells but deterministic given the seed). The other two axes are
physically meaningful: `epsilon` sets the equilibrium interface width and
the dominant unstable wavelength; `mean_init` sets the volume fraction of
each phase and, by symmetry, the topology of the late-time pattern
(droplets at `0.35` / `0.65`, bicontinuous strips at `0.50`).

## Numerical-stability strategy

1. **Bounded dynamics.** The Cahn-Hilliard equation has no advective CFL;
   the only stiffness comes from the fourth-order interface term, which
   the implicit Euler step on a bounded mesh handles unconditionally.
   The exponential `dt` ramp lets the early linear regime (small `dt`
   resolves the fastest growing mode) hand off smoothly to the
   coarsening regime (large `dt` skips ahead through the slow
   curvature-driven evolution).
2. **Per-step finite-check.** The driver verifies `np.isfinite(φ).all()`
   after every solve; non-finite values raise immediately so the SLURM
   job fails loudly rather than silently corrupting the trajectory.
3. **Per-run try/except.** On exception the driver writes a
   `run_XXXX.FAILED` sentinel JSON with the parameters and traceback,
   then exits non-zero. The SLURM array continues; a final
   `consolidate.py` step emits a report of any missing runs.
4. **Mesh determinism.** The Gmsh mesh is rebuilt from the same `.geo`
   string for every run with the same `(radius, cell_size)`; the cell
   ordering is therefore identical across `seed` values, so the IC
   noise pattern is the only source of run-to-run variability for fixed
   `(epsilon, mean_init)`.

## Computational details

- **Per-run wall.** ≈ 2 minutes on one scicore compute node, single
  process (FiPy is not MPI-parallelised on this mesh size). Mesh build
  ~2 s, solve ~107 s for ~700 implicit-Euler steps, resample ~1 s.
- **Total compute.** ≈ 2 core-hours for the full 48-run ensemble.
- **Solver precision.** `float64` throughout the simulation; output
  fields are downcast to `float32` at resample time.
- **Cluster.** sciCORE @ Universität Basel, `scicore` partition,
  `6hours` QoS, Easybuild `foss/2024a` +
  `HDF5/1.14.5-gompi-2024a` toolchain (the FiPy import path requires
  a system MPI library to be present, even though the solver itself is
  serial here).

## End-to-end data flow

```
# Login-node (need PyPI access).
uv sync --project datagen

# Emit per-run JSON configs + manifest.
uv run --project datagen python -m datagen.cahn_hilliard.scripts.generate_sweep

# Full 48-run sweep (one trajectory per array task, no MPI).
sbatch datagen/cahn_hilliard/slurm/sweep.sbatch

# Consolidate per-run Zarrs into the final dataset.zarr.
DATASET_ROOT=/scicore/home/dokman0000/GROUP/PDEDatasets/SphericalPDEs/cahn-hilliard-sphere
uv run --project datagen python -m datagen.scripts.consolidate \
    --processed $DATASET_ROOT/processed/ \
    --manifest  $DATASET_ROOT/manifest.json \
    --out       $DATASET_ROOT/dataset.zarr
```
